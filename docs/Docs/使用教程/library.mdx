# 快速接入 ​

## 各模型域名替换规则 ​

| 模型供应商 | 原域名 | 替换为 | 备注 |
|---|---|---|---|
| OpenAI | [https://api.openai.com/v1](https://api.openai.com/v1) | [https://api.openai-proxy.org/v1](https://api.openai-proxy.org/v1) | 一般需要加/v1 |
| Anthropic | [https://api.anthropic.com](https://api.anthropic.com) | [https://api.openai-proxy.org/anthropic](https://api.openai-proxy.org/anthropic) | 添加/anthropic后缀 |
| Gemini | [https://generativelanguage.googleapis.com](https://generativelanguage.googleapis.com) | [https://api.openai-proxy.org/google](https://api.openai-proxy.org/google) | 添加/google后缀 |

## 协议选择 ​

GPT API Proxy平台很多模型支持多种协议调用，比如Claude/Gemini，上述用法为原生协议域名，使用OpenAI兼容协议调用时，统一使用OpenAI的用法。

:::tip

OpenAI协议兼容由各供应商官方提供，属于官方的Beta功能，不排除未来官方废弃这种用法的可能，官方废弃后GPT API Proxy不再支持这种用法，GPT API Proxy不保证这种用法的稳定性，建议尽量使用原生协议。
:::

### 参考 ​

> 注意一定要替换为我们的api base和api key，差一个都是不对的。

```shell
curl https://api.openai-proxy.org/v1/chat/completions \
-H "Content-Type: application/json" \
-H "Authorization: Bearer sk-xxxxx" \
-d '{
    "model": "gpt-3.5-turbo",
    "messages": [{"role": "user", "content": "Hello!"}]
}'
```

:::warning

注意配置api_base时，应该加上一个/v1的后缀，而不是只有域名，要不然会报404错误。


部分过时的文档会给出带engine字段的示例，这个字段已经被废弃了，直接调openai可以调通，因为openai做了历史包袱兼容，平台是不支持这类被废弃的接口格式的。
:::
